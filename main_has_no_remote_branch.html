Felmeddelande 1 – "main has no remote branch"

The branch "main" has no remote branch. Would you like to publish this branch?
Vad det betyder: Din lokala branch main är inte kopplad till någon branch på GitHub ännu. Git vet inte vart den ska pusha.

När uppstår det?

Vanligast när du har initierat ett nytt lokalt repo med git init men aldrig pushat till GitHub tidigare. Din dator har koden – GitHub har ingenting.

✅ Lösning – klicka OK i Cursor-dialogen, eller kör i PowerShell:
git push -u origin main

Flaggan -u sätter upp kopplingen ("upstream") en gång för alla. Nästa gång räcker det med git push.

Felmeddelande 2 – "Can't push refs to remote. Try running Pull first"

Can't push refs to remote. Try running "Pull" first to integrate your changes.
Vad det betyder: GitHub-repot har commits som din lokala kopia saknar. Git vägrar skriva över dem – du måste hämta hem dem först.

Vanliga orsaker

  • GitHub skapade automatiskt en fil (t.ex. README eller .gitignore) när du satte upp repot
  • GitHub Pages-aktiveringen lade till en commit på distans
  • Någon annan (eller du själv via GitHub.com) redigerade en fil direkt på GitHub
✅ Lösning – kör i PowerShell:
git pull --rebase origin main
git push

--rebase lägger dina lokala commits "ovanpå" de inkomna ändringarna utan att skapa en extra merge-commit. Sedan pushar du som vanligt.

⚠️ Om pull skapar konflikter: Git markerar de drabbade filerna med <<<<<<<-markeringar. Öppna filen, välj vilken version du vill behålla, spara, och kör sedan git add <fil> följt av git rebase --continue.

Hela processen – från ny mapp till GitHub Pages

Det här är den kompletta sekvensen när du startar ett nytt projekt och vill ha det på GitHub med live-sidor via Pages.

Steg A Förbered lokalt repo

  1. Skapa eller öppna projektet i Cursor
  2. Öppna PowerShell-terminalen och navigera till projektmappen:
    cd "C:\Users\kentl\OneDrive\AI\Claude\Ovrigt"
  3. Initiera ett git-repo i mappen:
    git init

Steg B Skapa repo på GitHub.com

  1. Gå till github.com → New repository
  2. Ge repot ett namn (t.ex. Ovrigt)
  3. Lämna repot tomt – lägg INTE till README, .gitignore eller licens nu
    Om GitHub lägger till filer direkt får du "Can't push"-felet direkt
  4. Klicka Create repository – kopiera HTTPS-URL:en

Steg C Koppla ihop lokalt ↔ GitHub

  1. Lägg till remoten i PowerShell:
    git remote add origin https://github.com/kentlundgren/Ovrigt.git
  2. Kontrollera att det ser rätt ut:
    git remote -v

Steg D Lägg till filer och committa

  1. Staged alla filer du vill ha med:
    git add index.html
git add kalkylator.html
    eller lägg till allt på en gång: git add .
  2. Skapa en commit:
    git commit -m "Första versionen av projektet"

Steg E Pusha till GitHub (första gången)

  1. Pusha och sätt upp kopplingen:
    git push -u origin main
  2. Verifiera på github.com att filerna syns

Steg F Aktivera GitHub Pages

  1. Gå till ditt repo på GitHub.com → Settings → Pages
  2. Under Source: välj Deploy from a branch
  3. Branch: main  /  Folder: / (root)
  4. Klicka Save
  5. Vänta 2–5 minuter – sedan är sidorna live på:
    https://<användarnamn>.github.io/<reponamn>/
    Exempel: https://kentlundgren.github.io/Ovrigt/Hemma/laddboxar/index.html

Normalt arbetsflöde – efter att repot är uppsatt

När kopplingen väl är etablerad (efter Steg E ovan) är det bara dessa tre kommandon varje gång du vill spara och publicera ändringar:

git add .                          # Lägg till alla ändrade filer
git commit -m "Beskriv ändringen"  # Spara en ögonblicksbild
git push                           # Skicka upp till GitHub
Tips: Kör git status när som helst för att se vilka filer som är ändrade, stagade eller ännu inte tillagda.
